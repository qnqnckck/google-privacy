# Google Privacy Repo Rules

## Purpose
- This project exists only to publish a public privacy policy page for app store submission.
- Keep it static, simple, and publicly hostable.

## Scope
- Include only privacy-policy publishing assets and minimal supporting docs.
- Do not add application runtime code or unrelated tooling.
- Prefer plain HTML/CSS and Markdown over frameworks.

## Content Rules
- Keep policy language concrete and aligned with verified app behavior.
- Name the app explicitly and avoid claiming features the app does not have.
- Do not hardcode secrets or personal data.

## Verification
- Verify generated files are readable as static content.
- Verify GitHub Pages configuration steps are documented or applied when requested.
